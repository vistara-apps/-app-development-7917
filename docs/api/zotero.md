# Zotero Integration API

ScholarSift integrates with Zotero to provide seamless reference management capabilities. This document describes the Zotero integration and how it's used within the application.

## Overview

The Zotero integration allows users to:

1. Import papers from their Zotero libraries
2. Export summaries to Zotero as notes
3. Sync bibliographic data between ScholarSift and Zotero

## Authentication

ScholarSift uses OAuth to authenticate with Zotero:

1. User initiates Zotero connection in ScholarSift
2. User is redirected to Zotero for authorization
3. Zotero redirects back to ScholarSift with an authorization code
4. ScholarSift exchanges the code for an access token
5. The access token is used for all subsequent API calls

## API Endpoints

### Connect to Zotero

```
GET /api/v1/zotero/connect
```

Initiates the OAuth flow with Zotero.

### Get User Libraries

```
GET /api/v1/zotero/libraries
```

Returns all libraries accessible to the user.

#### Response

```json
{
  "libraries": [
    {
      "id": "string",
      "name": "string",
      "type": "user | group"
    }
  ]
}
```

### Get Library Items

```
GET /api/v1/zotero/libraries/:libraryId/items
```

Returns items from a specific library.

#### Query Parameters

- `q`: Search query
- `limit`: Maximum number of items to return
- `start`: Starting index for pagination

#### Response

```json
{
  "items": [
    {
      "key": "string",
      "version": "number",
      "title": "string",
      "creators": [
        {
          "firstName": "string",
          "lastName": "string",
          "creatorType": "string"
        }
      ],
      "date": "string",
      "url": "string",
      "DOI": "string",
      "itemType": "string"
    }
  ],
  "total": "number"
}
```

### Export Summary to Zotero

```
POST /api/v1/zotero/export
```

Exports a summary to Zotero as a note attached to an item.

#### Request

```json
{
  "summaryId": "string",
  "libraryType": "user | group",
  "libraryId": "string",
  "itemKey": "string"
}
```

#### Response

```json
{
  "success": "boolean",
  "note": {
    "key": "string",
    "version": "number"
  }
}
```

### Import Paper from Zotero

```
POST /api/v1/zotero/import
```

Imports a paper from Zotero and generates a summary.

#### Request

```json
{
  "libraryType": "user | group",
  "libraryId": "string",
  "itemKey": "string"
}
```

#### Response

```json
{
  "summaryId": "string",
  "title": "string",
  "summary": "string",
  "keyFindings": ["string"]
}
```

## Zotero API Usage

ScholarSift uses the following Zotero API endpoints:

- `/users/{userID}/collections`: Get user collections
- `/users/{userID}/items`: Get user items
- `/users/{userID}/items/{itemKey}`: Get a specific item
- `/users/{userID}/items/{itemKey}/children`: Get item children (notes, attachments)
- `/groups/{groupID}/collections`: Get group collections
- `/groups/{groupID}/items`: Get group items

## Data Mapping

### Zotero Item to ScholarSift Paper

```javascript
function mapZoteroItemToPaper(item) {
  return {
    title: item.title,
    url: item.url || (item.DOI ? `https://doi.org/${item.DOI}` : null),
    authors: item.creators.map(c => `${c.firstName} ${c.lastName}`).join(', '),
    year: item.date ? new Date(item.date).getFullYear() : null,
    doi: item.DOI
  };
}
```

### ScholarSift Summary to Zotero Note

```javascript
function formatSummaryAsZoteroNote(summary) {
  return `
    <h1>${summary.title}</h1>
    <p><strong>Source:</strong> <a href="${summary.url}">${summary.url}</a></p>
    <h2>Summary</h2>
    <p>${summary.summary}</p>
    <h2>Key Findings</h2>
    <ul>
      ${summary.keyFindings.map(finding => `<li>${finding}</li>`).join('')}
    </ul>
    <hr>
    <p><em>Generated by ScholarSift on ${new Date(summary.dateCreated).toLocaleDateString()}</em></p>
  `;
}
```

## Error Handling

Zotero integration errors are handled with specific error codes:

- `ZOTERO_AUTH_ERROR`: Authentication errors
- `ZOTERO_API_ERROR`: API errors
- `ZOTERO_RATE_LIMIT`: Rate limit exceeded
- `ZOTERO_ITEM_NOT_FOUND`: Item not found

## Rate Limiting

Zotero API has rate limits that ScholarSift respects:

- 300 requests per minute
- Exponential backoff for retries

## Security Considerations

- OAuth tokens are stored securely
- User data is only accessed with explicit permission
- API keys are never exposed to clients
